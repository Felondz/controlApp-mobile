# 🤝 Guía de Contribución

¡Gracias por tu interés en contribuir a ControlApp! Buscamos mantener un estándar alto de calidad y seguridad.

## 📋 Proceso de Pull Request

1. **Fork & Clone**: Crea tu propia copia del repositorio.
2. **Branch**: Crea una rama descriptiva (`feat/nueva-funcionalidad` o `fix/bug-login`).
3. **Code**: Implementa tus cambios siguiendo nuestros estándares (PSR-12 para PHP, componentes funcionales para React).
4. **Test**: Asegúrate de que `php artisan test` pase correctamente.
5. **Push & PR**: Envía tus cambios y abre un Pull Request describiendo detalladamente tu trabajo.

## 🔒 Seguridad
Si descubres una vulnerabilidad de seguridad, por favor **NO** abras un issue público. Reportalo directamente al equipo de mantenimiento o vía email privado.

## 🎨 Estándares de Diseño
- Utilizamos **TailwindCSS** con una configuración de colores semánticos.
- Respeta el modo oscuro en todos los componentes nuevos.
- Usa los componentes de UI existentes (`PrimaryButton`, `Card`, etc.) en lugar de crear nuevos desde cero.
