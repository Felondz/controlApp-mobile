# 🤝 Contributing Guide

Thank you for your interest in contributing to ControlApp! We aim to maintain a high standard of quality and security.

## 📋 Pull Request Process

1. **Fork & Clone**: Create your own copy of the repository.
2. **Branch**: Create a descriptive branch (`feat/new-feature` or `fix/login-bug`).
3. **Code**: Implement your changes following our standards (PSR-12 for PHP, functional components for React).
4. **Test**: Ensure `php artisan test` passes correctly.
5. **Push & PR**: Submit your changes and open a Pull Request detailing your work.

## 🔒 Security
If you discover a security vulnerability, please do **NOT** open a public issue. Report it directly to the maintenance team or via private email.

## 🎨 Design Standards
- We use **TailwindCSS** with a semantic color configuration.
- Respect dark mode in all new components.
- Use existing UI components (`PrimaryButton`, `Card`, etc.) instead of creating new ones from scratch.
